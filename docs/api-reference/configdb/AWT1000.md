# AWT1000 Configuration Database

Complete configuration reference for **AWT1000** firmware **v14.2.8**.

- **200** configuration variables
- **22** tables
- [:material-download: Download JSON Schema](AWT1000.schema.json){ .md-button }

!!! info "Using these variables"
    Read variables with `config.get()` and write them with `config.set()` within a
    configuration session. See [Configuration Variables](../config-variables.md) for
    usage examples. Table operations are documented in [Tables](../tables.md).

## VPN

!!! warning "Route push is not a security boundary"
    Network route push (`*_blxpush` variables) is a **routing convenience feature**, not
    a packet filter. It controls which local networks are reachable via the VPN tunnel by
    advertising routes, but it does not restrict traffic. To enforce access control, use the
    device's [packet filter](../../guides/packet-filter/index.md) or the Big-LinX
    server-side access policies.

### Permissions

#### `permissions` table

Configuration Permissions

Controls per-user read/write access to configuration variables and tables.
`configid` values: `0` = default permissions, `1-9999` = config variable ID,
table name (e.g., `"macgroups"`) = table access.

```python
cfg_session_id = dev.sess_start()
# Give user "operator" write access to config variable #42
dev.table_insert("permissions", cfg_session_id, ["operator", "42", "w"])
dev.sess_commit(cfg_session_id)
```
