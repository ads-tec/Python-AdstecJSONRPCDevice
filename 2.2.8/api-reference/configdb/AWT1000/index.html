    Using these variables

    Read variables with config.get() and write them with config.set() within a configuration session. See Configuration Variables for usage examples. Table operations are documented in Tables.

    VPN¶

    Route push is not a security boundary

    Network route push (*_blxpush variables) is a routing convenience feature, not a packet filter. It controls which local networks are reachable via the VPN tunnel by advertising routes, but it does not restrict traffic. To enforce access control, use the device's packet filter or the Big-LinX server-side access policies.

    Permissions¶

    permissions table¶

    Configuration Permissions

    Controls per-user read/write access to configuration variables and tables. configid values: 0 = default permissions, 1-9999 = config variable ID, table name (e.g., "macgroups") = table access.

    cfg_session_id = dev.sess_start()
# Give user "operator" write access to config variable #42
dev.table_insert("permissions", cfg_session_id, ["operator", "42", "w"])
dev.sess_commit(cfg_session_id)

    users table¶

    Configuration User Accounts

    Setting the password

    After inserting a user row, set the password in the same configuration session:

    cfg_session_id = dev.sess_start()
dev.table_insert("users", cfg_session_id,
    ["testuser", "", "", "1", "", "1970-01-01", "1970-01-01"])
dev.config_set(cfg_session_id, {
    "webpwd_user": "testuser",
    "webpwd_user_new": "SecurePassword123!",
    "webpwd_user_checked": "SecurePassword123!"
})
dev.sess_commit(cfg_session_id)
